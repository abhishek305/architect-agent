# Product Requirements Document
## Problem Statement
The problem we are trying to solve is to enable end-users to translate content within ContentStack, specifically in the entry sidebar location during the entry creation phase, without requiring any API configuration or paid tools.
## Goals & Success Metrics
The goal of this feature is to provide a basic, yet functional, translation capability that meets the requirements of in-context translation, no API configuration, and availability during the entry creation phase.
## Requirements
The requirements for this feature are:
* In-context translation: Translate content within the ContentStack UI, specifically in the entry sidebar location.
* No API configuration: No requirement for users to configure any APIs or paid tools.
* Entry creation phase: The translation feature should be available during the entry creation phase.
## User Stories
As a user, I want to be able to translate content in the entry sidebar location during the entry creation phase, so that I can create content in multiple languages without having to configure any APIs or paid tools.
## Acceptance Criteria
The acceptance criteria for this feature are:
* The translation feature is available in the entry sidebar location during the entry creation phase.
* The translation feature does not require any API configuration or paid tools.
* The translation feature provides accurate and reliable translations.
